SassQwatch

The Sass Query Watcher.

GitHub NPM

Sass ›› Javascript

We often use Sass to setup breakpoint variables for solid responsive designs. But our designs become fractured when we start hard-coding breakpoint values in our Javascript. SassQwatch links up your Sass and your JS for truly seamless responsive development.

Super Helpful Methods

SassQwatch has a bunch of really handy methods that help you write Javascript in a similar way to writing media queries in CSS. Check out the API docs below for a full list of what SassQwatch can do.

Responsive Images

There is nothing more expensive for a browser than trying to download massive images. SassQwatch offers an elegant way of serving up appropriate image sizes that are synced up with your breakpoints. Checkout out the demo below.

Sass/SCSS

$breakpoints: (
  small:    480px,
  medium:   768px,
  large:    1024px
);

Javascript

var sassqwatch = require('sassqwatch');

sassqwatch.min('medium', function() {
  console.log('This is above "medium"!');
});

Setup

You have two options. If you're already using NPM and Browserify, just npm install it and require it in your modules. If you're developing the old fashioned way, download the file from the repo and include it in the bottom of your document. In order to use it at that point you need to require it using CommonJS syntax.

NPM & Browserify

Command Line

              npm install --save sassqwatch

Javascript

var sassqwatch = require('sassqwatch');
sassqwatch.onChange(doSomethingSweet);

Manual

Download the script from the SassQwatch repo

<script src="js/sassqwatch.min.js"></script>
<script>
  var sassqwatch = require('sassqwatch');
  sassqwatch.onChange(doSomethingSweet);
</script>

Then setup your SCSS, Sass, LESS, or plain ole CSS to expose the breakpoints. We have provided a SassQwatch mixin to help you set everything up. Basically SassQwatch listens for media query changes on the head element, so you need to create a media queries for each breakpoint on that element and expose the name on the font-family property. Then on the font-family property of the title element you need to setup of a list of the breakpoints in order.

@media screen and (max-width: 479px) {
  head {
    font-family: "tiny";
  }
}
@media screen and (min-width: 480px) {
  head {
    font-family: "mini";
  }
}
@media screen and (min-width: 600px) {
  head {
    font-family: "small";
  }
}
@media screen and (min-width: 768px) {
  head {
    font-family: "medium";
  }
}
@media screen and (min-width: 960px) {
  head {
    font-family: "large";
  }
}
 
title {
  font-family: "tiny, mini, small, medium, large";
}

API

indicates a chainable method

properties

.breakpoints

An array of the names of the breakpoints in order.

console.log(sassqwatch.breakpoints);
// ['mq-mini', 'mq-small', 'mq-medium', 'mq-large', 'mq-xlarge']'

.current

The name of the current breakpoint.

console.log(sassqwatch.current);
// mq-xlarge

methods

.onChange( callback )

  • callback (function): the callback function to call when the breakpoint changes

Fires a callback on every breakpoint change. The callback is provided the name of the new breakpoint and the name of the previous breakpoint.

sassqwatch.onChange(function (newBreakpoint, oldBreakpoint) {
  console.log('breakpoint switched to ' + newBreakpoint + ' from ' + oldBreakpoint);
});

.min( breakpoint, callback, fireOnce )

  • breakpoint (string): the name of the breakpoint to check against
  • callback (function): the callback function to call
  • fireOnce (boolean): causes the callback to only be fired the first time the condition is met. If set to false the callback will be fired on every breakpoint change where the condition is met. (Default: true)

A convenience method that functions similarly to a `min-width` breakpoint in CSS. The callback is fired on the specified breakpoint as well as any breakpoint that is above it. The callback is provided the name of the new breakpoint and the name of the previous breakpoint.

sassqwatch.min('mq-medium', function (newBreakpoint, oldBreakpoint) {
  console.log('now min mq-medium');
});

.max( breakpoint, callback, fireOnce )

  • breakpoint (string): the name of the breakpoint to check against
  • callback (function): the callback function to call
  • fireOnce (boolean): causes the callback to only be fired the first time the condition is met. If set to false the callback will be fired on every breakpoint change where the condition is met. (Default: true)

A convenience method that functions similarly to a `max-width` breakpoint in CSS. The callback is fired on any breakpoint that is below it and is not inclusive. The callback is provided the name of the new breakpoint and the name of the previous breakpoint.

sassqwatch.max('mq-medium', function (newBreakpoint, oldBreakpoint) {
  console.log('now max mq-medium');
}, false);

.only( breakpoint, callback )

  • breakpoint (string): the name of the breakpoint to check against
  • callback (function): the callback function to call

A convenience method that fires a callback only on a specified breakpoint. The callback is provided the name of the previous breakpoint.

sassqwatch.only('mq-medium', function (oldBreakpoint) {
  console.log('now on mq-medium');
});

.query( type, breakpoint, callback, fireOnce )

  • type (string): "min", "max", or "only"
  • breakpoint (string): the name of the breakpoint to check against
  • callback (function): the callback function to call
  • fireOnce (boolean): causes the callback to only be fired the first time the condition is met. If set to false the callback will be fired on every breakpoint change where the condition is met. Only applies to 'min' and 'max' types. (Default: true)

Fires a callback when the current breakpoint is min, max, or only the specified breakpoint. If checking for "min" or "max" then the callback receives the name of the new breakpoint. If checking for "only" the callback receives the name of the old breakpoint.

sassqwatch
  .query('min', 'mq-medium', function (newBreakpoint, oldBreakpoint) {
    console.log('now min mq-medium');
  })
  .query('only', 'mq-xxlarge', function (oldBreakpoint) {
    console.log('now on mq-xxlarge');
  });

.isBelow( breakpoint, inclusive )

  • breakpoint (string): the breakpoint to check against
  • inclusive (boolean): whether or not to include the breakpoint in the check; <= instead of <. (Default: false)

Returns `true` if the current breakpoint is below a specified breakpoint, and `false` otherwise.

if ( sassqwatch.isBelow('mq-large') ) {
  console.log('breakpoint is below mq-large.');
}

.isAbove( breakpoint, inclusive )

  • breakpoint (string): the breakpoint to check against
  • inclusive (boolean): whether or not to include the breakpoint in the check; >= instead of >. (Default: true)

Returns `true` if the current breakpoint is above a specified breakpoint, and `false` otherwise.

if ( sassqwatch.isAbove('mq-large') ) {
  console.log('breakpoint is above mq-large.');
}

.matches( breakpoint )

  • breakpoint (string): the breakpoint to check against

Returns `true` if the current media query matches a specified media query, and `false` otherwise.

if ( sassqwatch.matches('mq-large') ) {
  console.log('breakpoint is mq-large.');
}
